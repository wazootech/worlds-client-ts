# Domain model

This documentation standardizes the core vocabulary and systemic invariants
governing the client-side edge semantic environments.

## Core glossary

### Graph store

The transient in-memory representation of RDF facts, currently backed by an
`N3.Store`. It facilitates high-speed, queryable operations without recurring
network hop penalties.

### Hydration

The bootstrapping process that reconstructs the Graph Store's ephemeral state by
deserializing master relational tuples from stable external persistence
(currently LibSQL) into reactive memory nodes.

### Synchronization

The process by which atomic mutation queues (inserts and deletes) generated by
application interactions are serialized and reliably persisted to authoritative
storage providers.

### Reactive client

The primary unified orchestration layer fusing a synchronized Graph Store, a
SPARQL engine, and fuzzy search indexes into a cohesive, ACID-compliant runtime
facade.

### Idempotency guard

An optimized interceptor layer evaluating existing Graph Store state prior to
emitting mutation signals. It secures compute efficiency by suppressing
redundant external API interactions and precluding database duplication.

### Hybrid search

A robust retrieval facility supporting native fusion of Vector Semantic
similarity and Keyword Full-Text Search. It provides automatic graceful
degradation to sustain query availability during intermittent upstream API
outages, and natively supports fully vectorless keyword-only deployments when
embedding engines are omitted.

### Textual literal

A specialized subset of RDF Literals that physically represent textual strings
(including explicit `xsd:string` and localized `rdf:langString` tags, as well as
untyped literals). It serves as the canonical filter for semantic indexing,
explicitly excluding structured primitives (numeric values, booleans,
timestamps) to suppress high-frequency noise and optimize search space
relevance.
